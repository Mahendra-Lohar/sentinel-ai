# Contributing to Sentinel AI

First off, thank you for considering contributing to Sentinel AI! It's people like you that make Sentinel AI such a great platform for the DevOps and SRE community.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for Sentinel AI. 
- Use the GitHub Issue Tracker.
- Describe the bug clearly. Provide logs, environment details, and screenshots if applicable.
- Explain the expected behavior vs. actual behavior.

### Suggesting Enhancements
- Open an Issue labeled `enhancement`.
- Provide a clear, detailed explanation of the feature.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`npm test`).
4. Make sure your code lints (`npm run lint`).
5. Issue that pull request!

## Local Development
Please see the `README.md` for instructions on setting up the local Docker-Compose environment for testing.

## Code Style
- Use ES6+ syntax.
- Follow the existing Prettier/ESLint configurations.
- Use explicit imports and avoid wildcard exports where possible.
